# Vibe Coding

使用`Harness Engineering`理念驾驭AI

- 1、上下文工程 (工作手册/工程目录)
- 2、架构约束  （规定工程架构/约束）
- 3、反馈循环  （AI自身生成测试用例）
- 4、垃圾回收  （扫描文档/代码匹配度）

## Awesome Prompt

### 第一性原理

``` txt
以第一性原理！从原始需求和问题本质出发，不从惯例或模版出发
1、不要假设我清楚自己想要什么。动机或目标不清晰时，停下来讨论。
2、目标清晰但路径不是最短的，直接告诉我并建议更好的办法。
3、遇到问题追原因，不打补丁。每个决策都要能回答“为什么”
4、输出说重点，砍掉一切不改变决策的信息。
```
